Ensemble 新手教程

Ensemble 是一个桌面多 Agent 工作台。你可以把 Claude、OpenAI、OpenAI Compat 和 Codex CLI agent 放在同一个 workspace 里，让它们通过 peer_send / peer_query 协作。

快速开始

  1. 安装 Windows 版 Ensemble_0.0.14_x64-setup.exe。
  2. 打开左侧 Providers，添加至少一个可用 provider。
  3. 点击左侧 Agent 区域的 +，选择 provider 和 model，创建第一个 agent。
  4. 在聊天框输入任务，按 Enter 发送。
如果你使用本地 Claude 或 Codex，需要先在系统终端完成对应 CLI 登录：claude / codex login。

配置 Provider

商业使用建议：稳定生产通知链路优先选 openai-local / openai-compat / anthropic-local。Codex CLI 已支持 peer tools，但仍依赖用户本机 Codex CLI 版本和登录状态。

创建 Agent

  • Name：给 agent 一个可被同伴识别的名字，例如 Reviewer。
  • Provider / Model：决定它使用哪个 runtime。
  • System Prompt：定义角色、职责和交付标准。
  • Codex Workspace：仅 Codex provider 使用，建议指向实际项目目录。

你可以创建多个 agent，并在同一个窗口内切换、并行运行、互相发送消息。

Agent 互相通知

Agent 间协作主要依赖两个工具：

  • peer_send：向另一个 agent 发送新消息，可触发对方继续工作。
  • peer_query：只读取另一个 agent 最近对话，不触发对方运行。

常用模式：

  • raw：普通通知或提问。
  • continue：把当前工作交给对方继续。
  • review：请对方审查你的输出。
  • fork：请对方独立给出另一版方案。

Codex CLI 使用说明

  1. 先安装并登录 Codex CLI：npm i -g @openai/codex，然后运行 codex login。
  2. 在 Ensemble 的 Provider 面板添加 OpenAI -> Codex CLI。
  3. 为 Codex agent 设置合适的 workspace 和 sandbox。
Codex CLI 仍是本机 CLI 运行态。它可能影响用户正在使用的外部 Codex CLI 会话；如果外部会话被打断，可以用 codex resume 恢复。Ensemble 已通过 stdio MCP bridge 支持 Codex agent 调用 peer_send / peer_query，但发布前仍应跑 smoke 测试确认当前 Codex CLI 版本没有改变 MCP 行为。

MCP 和 Skills

MCP 面板可以添加外部工具服务，支持 stdio、http、sse。Skills 使用 SKILL.md，适合沉淀可复用工作流。

{
  "command": "npx",
  "args": ["-y", "some-mcp-server"],
  "env": { "TOKEN": "..." }
}

Agent 可通过 skill_list 查看技能，通过 skill_invoke 加载指定技能。

常见问题

Codex agent 不能互相通知？

确认 Codex CLI 已登录、Provider 是 openai-codex、agent 名称拼写正确。当前版本通过 stdio MCP bridge 暴露 peer tools；如果失败，优先运行 Codex peer smoke 测试定位。

目标 agent 收到消息但运行失败？

这说明 peer_send 已送达，失败点在目标 agent 的 provider 或模型连接。检查目标 provider 的 key、baseUrl 和 model。

安装新版后仍是旧行为？

关闭所有 Ensemble 窗口后重新安装。安装器会替换桌面壳和 core sidecar，两者必须来自同一版本。
