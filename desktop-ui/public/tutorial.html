Ensemble 新手教程

Ensemble 是一个桌面多 Agent 工作台。你可以把 Claude、OpenAI、OpenAI Compat 和 Codex CLI agent 组织成 team，在同一个 workspace 里通过 peer_send / peer_query 协作。

快速开始

  1. 安装当前平台版本：Windows x64 为 Ensemble_0.0.27_x64-setup.exe；macOS Apple Silicon / Intel 为 Ensemble_0.0.26_aarch64.dmg / Ensemble_0.0.26_x64.dmg。
  2. 打开左侧 Providers，添加至少一个可用 provider。
  3. 点击 + team 创建团队，给每个成员选择独立的 provider、model 和角色 prompt；也可以先用 + 创建单个 agent。
  4. 在聊天框输入任务，按 Enter 发送。
Windows 与 macOS 通过平台隔离 manifest 更新：客户端只选择自己系统和架构对应的安装包。缺失某个平台条目时，客户端会抑制错误平台更新提示。

创建 Team

  • Team name / purpose：描述团队目标；这个目的会作为 TEAM CONTEXT 注入成员 prompt。
  • Members：每个成员单独设置角色名、provider、model 和 system prompt。
  • 跨模型协作：常见组合是 Claude 做方案、GPT 做审阅、Codex 在项目目录里执行代码任务。
  • 编辑生效：修改团队目的、成员角色或团队归属时，Ensemble 会清理 resume 指针，让下一轮使用新 prompt，同时保留聊天历史。

单 agent 仍可直接创建；团队只是把一组 agent 的共同目标、成员列表和跨模型协作关系固定下来。

Cloud beta

Cloud beta 通过 Ensemble server / cloud infrastructure 提供账号、workspace 同步、网页工作区、远程发送和部分 agent 设置入口。登录网页后，桌面在线时可以向本地 agent 发送消息并查看结果；模型调用、本地文件访问、CLI 登录态、MCP 和权限审批仍由桌面端执行。

  • 桌面端登录 Cloud 后，可以把当前本地 workspace 的 agents、teams 和消息批量同步到云端。
  • 官网 /workspace.html 可查看同步后的 workspace，并在桌面在线时向某个本地 agent 发送远程消息、查看返回结果。
  • 远程设置支持 name、model、providerId、permission、sandbox、reasoning、Codex workspace、teamId 和 system prompt 等字段；设置请求仍由在线桌面端应用。
桌面端是执行端，Cloud beta 是账号、同步、网页工作区和远程通道。桌面离线时，网页仍可查看已同步内容；新的远程发送和设置应用会等桌面重新在线。

Agent 互相通知

Agent 间协作主要依赖两个工具：

  • peer_send：异步向另一个 agent 发送新消息；目标正在运行时会排队，发送方不等待对方回复。
  • peer_query：只读取另一个 agent 最近对话，不触发对方运行。

常用模式：

  • raw：普通通知或提问；默认只发送 message 本身。
  • continue：把当前工作交给对方继续；默认附带发送方最近关键输出。
  • review：请对方审查你的输出；默认附带来源上下文。
  • fork：请对方独立给出另一版方案；默认附带来源上下文作为参考。

includeSource 默认为 auto：raw 只送消息，continue / review / fork 会附带有界 source-output。批量 queued peer send 会合并投递；interrupt=true 必须提供 interruptReason，只适合紧急安全阀，不是普通通知方式。

Codex CLI 使用说明

  1. 先安装并登录 Codex CLI：npm i -g @openai/codex，然后运行 codex login。
  2. 在 Ensemble 的 Provider 面板添加 OpenAI -> Codex CLI。
  3. 为 Codex agent 设置合适的 workspace、sandbox 和 reasoning effort。
Codex 是本机 CLI subprocess，不是旧的 SDK 直连。Ensemble 使用隔离的 CODEX_HOME 写运行配置，支持 read-only / workspace-write / danger-full-access 三档 sandbox；peer_send / peer_query 通过本地 stdio MCP bridge 暴露给 Codex。

Agent 设置中的 Reasoning / thinking mode 可按 agent 继承或覆盖。改模型、团队、角色、sandbox 等配置时，Ensemble 会尽量保留用户的 reasoningEffort / sandboxMode 等显式设置，只清理必须失效的 resume 指针。

MCP 和 Skills

MCP 和 Skills 是两类能力，不要混在一起理解：

  • MCP：外部工具服务，支持 stdio、http、sse transport。
  • Skills：SKILL.md 指令包，适合沉淀可复用工作流；来源包括 project、ensemble、user 和 system。
{
  "command": "npx",
  "args": ["-y", "some-mcp-server"],
  "env": { "TOKEN": "..." }
}

左侧 Skills 面板可以新建、编辑和重扫 ensemble 管理的 skill。Agent 可通过 skill_list 查看技能，通过 skill_invoke 加载指定技能；每个 agent 还支持 forced、disabled 和 auto activation 状态。

用量统计

使用 /cost 或顶部用量入口打开 Usage Stats。面板按 agent、model、provider、日期展示 input / output / cache read / cache write token 和成本，并支持 CSV 导出。

用量记录是本地审计数据。Codex 通过 ChatGPT 订阅运行时，统计会记录 token，但不会把订阅型 turn 按 API 单价重复计费。

常见问题

Codex agent 不能互相通知？

确认 Codex CLI 已安装并登录、Provider 是 openai-codex、agent 名称拼写正确。当前版本通过 stdio MCP bridge 暴露 peer tools；如果失败，优先运行 Codex peer smoke 测试定位。

目标 agent 收到消息但运行失败？

这说明 peer_send 已送达，失败点在目标 agent 的 provider 或模型连接。检查目标 provider 的 key、baseUrl 和 model。

聊天流式输出会不会自动跟随？

会。流式文本增长、新消息、running/status 变化、回焦和空态 observer 重绑都会触发自动跟随；如果你手动滚动离开底部，Ensemble 会尊重当前位置。

安装新版后仍是旧行为？

关闭所有 Ensemble 窗口后重新安装。安装器会替换桌面壳和 core sidecar，两者必须来自同一版本。
